# Skill: betaform Voice & Brand

Dieses Skill gilt für alle Content-Outputs: Cold Emails, LinkedIn-Posts (Personal + Company), Bild-Prompts.
Es basiert auf der echten Website, den Blog-Artikeln und Oles nachgewiesenem Schreibstil.

---

## Wer betaform ist (für Emails + Posts)

**Zwei Menschen, Hamburg:**
- **Ole Beekmann** — Produkt & UX. Designer der auch baut. Denkt in Flows, Komponenten, Nutzerverhalten.
- **Carina** — Brand Design.

**Was betaform macht:**
- UX/Product Design + MVP-Entwicklung für komplexe B2B-Produkte
- Spezialisierung: B2B SaaS, Plattformen, Enterprise & Government Software
- Von UX-Strategie bis lauffähigem MVP — Klarheit, Struktur, Wirkung
- Vibe Coding: Cursor + Claude + v0 → Designer der ohne Backend-Entwickler baut

**Was betaform NICHT macht:** Kampagnen, PR, Social Media-Management, Consumer-Produkte

**Warum betaform vs. große Agenturen:**
- Direkte Kommunikation: wer anruft, spricht mit wem umsetzt
- Kein zweites Liga-Team: jedes Projekt ist Hauptprojekt
- Tiefe schlägt Breite: B2B SaaS UX, Design Systems, Produktstrategie — seit Jahren, ausschließlich

## Tonalität: Personal vs. Company — GLEICHE STIMME, andere Perspektive

Ole hat definiert: Authentizität gilt für beide Profile.
Das bedeutet: Nicht zwei verschiedene Personas — eine Stimme, zwei Blickwinkel.

| | LinkedIn Personal (Ole) | LinkedIn Company (betaform) |
|---|---|---|
| Perspektive | Ich — persönliche Erfahrung, Meinung, Prozess | Wir — Studio-Sicht, Deliverables, Methodik |
| Ton | Norddeutsch-nüchtern, erste Person | Professionell aber direkt, kein Corporate-Speak |
| Themen | Was Ole gerade erlebt, lernt, ausprobiert | Was betaform liefert, wie betaform denkt |
| Hook | Persönliche Beobachtung, oft konträr | Konkretes Deliverable oder Erkenntnis aus Projekt |
| CTA | Echte Frage ("Wie macht ihr das?") | Konkreter nächster Schritt (Case ansehen, Gespräch) |
| Hashtags | #UX #AIDesign #betaform (genau 3) | Themenspezifisch, max. 3 |

**Wichtig:** Company-Posts klingen nicht wie eine andere Person. Sie klingen wie Ole, der über betaform spricht — direkt, ohne Corporate-Filter.

## Bildsprache — beide Profile

**Primärquelle für Farben, Tokens und Ästhetik: Skill `betaform/betaform-design`** (colors_and_type.css, README.md)

Kurzfassung: Warm Cream `#FAF9F6` Hintergrund, Aurora-Gradient aus 5 Pastelltönen (Dusty Rose `#f5cec6`, Peach `#ecd5ce`, Slate-Blue `#c8d5e4`, Lavender `#e8e4f6`, Sage `#d2e0d8`), betaform Orange `#F5943A` als Akzent. Söhne-Font, große Radii (24–36px), kein Stock-Foto-Stil.

### LinkedIn Personal (Ole)

Komposition: **Input → Pfeil → UI-Output**

```
Flat vector illustration. Background: warm cream #FAF9F6.
No humans, no laptops, no coffee, no stock photo elements.

Three elements left to right:
1. Empty input field with cursor "|" — white #FFFFFF, pill shape, subtle shadow
2. Gradient arrow right — lilac #C4A8F8 to salmon #F7C8C0
3. UI component — white card #FFFFFF, pill shape (border-radius 24px), subtle shadow

betaform orange #F5943A for active elements.
Max 1 word per label. Correct spelling. Quality: high.
No glow, no neon, no dark backgrounds.
```

UI-Komponente je Post-Thema:

| Thema | Rechte Komponente | Labels |
|---|---|---|
| Vibe Coding / KI-Workflow | Tab-Bar 2 Tabs | "Prompt" / "Code" |
| Design System / Tokens | Tab-Bar 2 Tabs | "Token" / "Style" |
| UX Research | Tab-Bar 2 Tabs | "Draft" / "Final" |
| Produktstrategie | Tab-Bar 2 Tabs | "Problem" / "Design" |
| Accessibility / QA | Tab-Bar 2 Tabs | "Check" / "Done" |
| Studio-Alltag | Tab-Bar 2 Tabs | "Input" / "Output" |
| GovTech / Enterprise | Tab-Bar 2 Tabs | "Daten" / "UI" |

### LinkedIn Company (betaform)

```
Clean portfolio-quality UI mockup. White or warm cream background.
Show one concrete deliverable: a real-looking B2B interface, dashboard, or design system component.
Flat, professional, readable labels (max 3 words). No glow, no electric blue.
Inspired by betaform.io aesthetic: soft colors, generous whitespace, large clean typography.
Quality: high.
```

Konkreter je Thema:
- Design System Post → zeige Komponenten-Grid oder Token-Übersicht
- Case Study Post → zeige UI-Screen des Projekts (abstrakt, kein echtes Logo)
- Methodik Post → zeige Wireframe-Struktur oder Flow-Diagramm

### Instagram (beide)
```
Authentic, warm studio aesthetic. Behind-the-scenes feel.
A screen in context, a sketchbook, design details, Hamburg surroundings.
No staged setups. Warm natural light. Real and human.
Quality: high.
```

## Cold Email — Tonalität & Beispiele

**Stimme:** Gleich wie Blog — direkt, norddeutsch-nüchtern, keine Warm-up-Sätze.
**Primärquelle Tonalität:** BET-226

**Struktur (50–120 Wörter):**
1. Hook: Trigger direkt benennen — keine Einleitung
2. Problem: konkret, eine Zeile
3. Referenz: ein Name, eine Zeile
4. CTA: eine Frage

### Was nie funktioniert
- "Ich schreibe Ihnen, weil ich auf Ihr Unternehmen aufmerksam geworden bin..."
- Bullet-Point-Listen eigener Leistungen
- Mehr als ein CTA
- Länger als 120 Wörter
- "Wir bieten..." als Einstieg
- Superlative ("erstklassig", "führend", "einzigartig")
